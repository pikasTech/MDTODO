# R23.2 编号列表渲染修复执行报告

## 问题描述

R23.2 要求编号和其他 markdown 语法能在非编辑模式下正常渲染，在非编辑模式下不需要过滤。

## 问题分析

在 `parser/index.ts` 的 `parseTask` 函数中，标题处理逻辑移除了列表标记：
```typescript
.replace(/^\s*[\-\*]\s+/gm, '')  // 移除列表标记
.replace(/^\s*\d+\.\s+/gm, '')  // 移除数字列表
```

这导致非编辑模式下无法显示编号列表格式。

## 修复方案

从 `parseTask` 函数中移除列表标记过滤代码，保留所有 markdown 语法格式，让 marked 库在渲染时正常处理。

## 修改的文件

1. **src/parser/index.ts** - 移除第273-274行的列表过滤代码

## 修复效果

- 非编辑模式下，任务内容的 `(1)`、`(2)`、`1.`、`2.`、`-` 等 markdown 语法将完整保留
- marked 库能正确渲染编号列表为 HTML
- 编辑模式下使用 `rawContent` 保持原始编辑体验

## 构建验证

```bash
npm run compile  # 编译成功
```
