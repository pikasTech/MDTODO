# R50.4 任务报告：按钮位置优化

## 任务目标
将是否完成的选择框和控制任务展开/折叠的两个按钮从最左侧移到链接状态的右侧，提高文本的空间利用率。

## 执行内容

### 1. 修改 TaskList.tsx

**修改位置1**：将 `expand-icon` 和 `task-checkbox` 从 `task-main-left` 中移除，移到 `task-id-wrapper` 内，位于 `link-status-icon` 右侧。

**修改位置2**：删除 `task-main-left` 中原来的 `expand-icon` 和 `task-checkbox` 元素。

关键代码变更：
- 原来：`task-main-left` 包含 `expand-icon` → `task-checkbox` → `task-content`
- 现在：`task-main-left` 直接包含 `task-content`，`expand-icon` 和 `task-checkbox` 在 `task-id-wrapper` 内紧跟在 `link-status-icon` 后面

### 2. 修改 TaskList.css

1. **`.task-id-wrapper`**：添加 `flex: 1` 和 `min-width: 0` 使其填充剩余空间并正确处理溢出
2. **`.expand-icon`**：改为 `display: inline-flex`，调整宽度和高度使其更紧凑
3. **`.task-checkbox`**：改为 `display: inline-flex`，调整尺寸使其与其他元素对齐

## 修改文件
- `src/webview/components/TaskList.tsx`
- `src/webview/components/TaskList.css`

## 验证结果
- 编译成功，无语法错误
- 展开/折叠按钮和完成选择框现在位于链接状态图标右侧
- 文本内容区域不再被左侧按钮占用空间
