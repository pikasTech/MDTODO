# R51.5.1 任务报告：筛选栏收起/展开遮挡功能修复

## 任务目标

修复 R51.5 实现的问题：筛选栏在收起/展开时仍然推挤下面的任务块，应该改为遮挡上方内容。

## 问题分析

R51.5 的实现使用了 `position: absolute` 来让筛选栏覆盖在内容上方，但问题在于：

1. **父容器缺少定位参照**：`.app` 容器没有设置 `position: relative`，导致 `filter-bar-container` 的绝对定位失效（会相对于视口定位而不是相对于父容器）

2. **绝对定位需要正确的参照**：只有当父容器设置了 `position: relative` 时，子元素的绝对定位才会相对于父容器边界

## 修复方案

给 `.app` 容器添加 `position: relative`：

```css
.app {
  display: flex;
  flex-direction: column;
  height: 100vh;
  background-color: #1e1e1e;
  color: #d4d4d4;
  padding: 0 0;
  position: relative; /* 【修复R51.5.1】为绝对定位的子元素提供定位参照 */
}
```

## 修改文件

- **src/webview/bundle.css**：
  - 给 `.app` 添加 `position: relative`

## 验证方法

1. 打开 MDTODO 插件
2. 验证筛选栏默认显示为收起状态
3. 鼠标移动到header区域，验证筛选栏展开
4. 观察筛选栏展开时是否遮挡了第一行的任务，而不是推挤任务内容下移
5. 鼠标离开header和筛选栏区域，验证筛选栏收起，任务列表恢复正常显示位置
