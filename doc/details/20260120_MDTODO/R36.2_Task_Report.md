# R36.2 报告路径格式修改执行记录

## 需求说明

将新建任务时自动生成的报告路径名从 `[Rxx](./details/{MDTODO}/Rxx_任务描述.md)` 改为 `[Rxx](./details/{MDTODO}/Rxx_Task_Report.md)`。

## 修改文件

`vscode-mdtodo/src/providers/webviewProvider.ts`

- `generateNewTaskContent` 方法：修改报告文件名格式

## 详细修改

### 修改前

```typescript
const reportPath = `./details/${fileName}/${taskId}_任务描述.md`;
```

生成的内容示例：
```
## R36

, 完成任务后将详细报告写入[R36](./details/20260120_MDTODO/R36_任务描述.md)。
```

### 修改后

```typescript
const reportPath = `./details/${fileName}/${taskId}_Task_Report.md`;
```

生成的内容示例：
```
## R36

, 完成任务后将详细报告写入[R36](./details/20260120_MDTODO/R36_Task_Report.md)。
```

## 效果说明

1. **新建主任务**：添加任务后，任务内容自动填充为：
   ```
   ## R36

   , 完成任务后将详细报告写入[R36](./details/20260120_MDTODO/R36_Task_Report.md)。
   ```

2. **新建子任务**：添加子任务后，任务内容自动填充为：
   ```
   ### R36.1

   , 完成任务后将详细报告写入[R36.1](./details/20260120_MDTODO/R36.1_Task_Report.md)。
   ```

3. 报告文件命名更简洁统一，使用英文 `Task_Report` 代替中文 `任务描述`。

## 测试验证

1. 点击"+ 添加任务"按钮
2. 验证新任务内容包含新的报告模板路径格式
3. 点击"+ 子任务"按钮
4. 验证子任务内容包含新的报告模板路径格式
