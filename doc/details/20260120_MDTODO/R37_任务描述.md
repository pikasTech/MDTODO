## 任务描述

增加一个打开原MDTODO的原md文件的功能。

## 执行记录

### 实现方案

在 MDTODO Webview 界面中添加"打开原文"按钮，点击后使用 Typora 打开当前加载的原始 MDTODO 文件。

### 修改内容

#### 1. webviewProvider.ts

添加 `handleOpenSourceFile` 方法用于处理打开原文文件的请求：

```typescript
case 'openSourceFile':
  // 【实现R37】打开原MDTODO文件
  await this.handleOpenSourceFile();
  break;

/**
 * 【实现R37】打开原MDTODO文件（使用 Typora 或 VSCode）
 */
private async handleOpenSourceFile(): Promise<void> {
  if (!this.currentFilePath) {
    vscode.window.showWarningMessage('未加载任何文件');
    return;
  }

  // 检查文件是否存在
  if (!fs.existsSync(this.currentFilePath)) {
    vscode.window.showErrorMessage(`文件不存在: ${this.currentFilePath}`);
    return;
  }

  console.log('[MDTODO] 打开原文件:', this.currentFilePath);

  // 使用 Typora 打开 md 文件（与 handleOpenLink 逻辑一致）
  await this.openWithTypora(this.currentFilePath);
}
```

#### 2. TaskList.tsx

添加 `handleOpenSourceFile` 处理函数：

```typescript
// 【实现R37】打开原MDTODO文件
const handleOpenSourceFile = () => {
  // 防抖：如果冷却中，不执行
  if (buttonCooldown) {
    console.log('[Webview] 打开原文按钮防抖，跳过重复点击');
    return;
  }
  sendMessage({ type: 'openSourceFile' });
  // 设置防抖状态
  setButtonCooldown(true);
  setCooldownMessage('原文');
  setTimeout(() => {
    setButtonCooldown(false);
    setCooldownMessage(null);
  }, BUTTON_COOLDOWN);
};
```

在 header-actions 中添加"打开原文"按钮：

```typescript
React.createElement('button', {
  className: `btn btn-secondary ${buttonCooldown ? 'disabled' : ''}`,
  disabled: buttonCooldown,
  onClick: handleOpenSourceFile,
  title: '在Typora中打开原文'
}, '打开原文'),
```

### 功能说明

1. **按钮位置**：位于 header 区域的"打开文件"按钮右侧
2. **打开方式**：默认使用 Typora 打开（与链接点击打开 md 文件的逻辑一致），如果 Typora 未安装则回退到 VSCode
3. **防抖保护**：与其他按钮一样具有 0.5 秒防抖保护，避免误触

### 测试方法

1. 在 VSCode 中打开一个 MDTODO 文件（如 20260120_MDTODO.md）
2. 点击 Webview 界面右上角的"打开原文"按钮
3. 验证文件是否在 Typora 中打开（如果 Typora 已安装）
