# R49.1 任务报告：时间戳命名修复

## 问题描述

R49 实现的时间戳命名方式没有生效，创建新任务时仍然使用 RXX_Task_Report.md 命名。

## 根因分析

**根本原因：VSCode 插件编译配置严重不匹配**

| 配置文件 | 配置项 | 期望路径 | 实际路径 |
|---------|-------|---------|---------|
| tsconfig.json | `outDir` | `out/` | `out/` ✓ |
| webpack.config.js (ext) | `output.path` | `out/` | `dist/` ✗ |
| package.json | `main` | `out/extension.js` | - |
| launch.json | `outFiles` | `out/**/*.js` | - |

**问题链路**：
1. `npm run compile` 执行 `tsc && webpack`
2. `tsc` 将 `src/extension.ts` 编译到 `out/extension.js`（无 webpack 注入）
3. `webpack` 将 `src/extension.ts` 编译到 `dist/extension.js`（有 webpack 注入）
4. VSCode 根据 `package.json` 的 `main` 字段加载 `out/extension.js`
5. 结果：加载的是**未注入 webpack 逻辑**的旧代码！

## 解决方案

修改 `webpack.config.js`，将 extension bundle 的输出路径从 `dist/` 改为 `out/`：

```javascript
// 修改前
output: {
  path: path.resolve(__dirname, 'dist'),
  filename: 'extension.js',
}

// 修改后
output: {
  path: path.resolve(__dirname, 'out'),
  filename: 'extension.js',
}
```

## 修复后的编译流程

```
npm run compile
  ├── tsc → out/ (仅类型检查，产物被 webpack 覆盖)
  └── webpack → out/extension.js + resources/bundle.js ✓
```

## 验证结果

```bash
# 编译输出
asset extension.js 39.4 KiB
asset bundle.js 86.9 KiB

# 验证时间戳代码存在于 out/extension.js
out/extension.js: ✓ 包含时间戳命名代码
```

## 使修改生效

- **调试模式**：重新加载 VSCode 窗口（Ctrl+Shift+P → "Reload Window"）
- **生产模式**：运行 `install.bat` 重新打包安装

## 修改文件

- `webpack.config.js` - 修改 extension bundle 输出路径为 `out/`
