# R53.1 任务报告：撤销 R53 修复，恢复代码块作为普通文本块的一部分

## 问题描述

R53 的修复方向完全错误。用户需要的是"代码块作为普通文本块的一部分渲染"，而不是将代码块从普通文本块中分离出去。

## 问题分析

**R53 的错误修改**：
- 在 `parseTextBlocks` 方法中添加了 `!inCodeBlock` 检查
- 导致代码块内容被完全跳过，不作为普通文本块的一部分
- 这违背了用户的需求

**正确行为**：
- 代码块内容应该包含在普通文本块中一起渲染
- 移除 R53 添加的所有 `!inCodeBlock` 检查

## 修复方案

撤销 R53 的修改，移除所有 `!inCodeBlock` 检查：

1. 第 159 行：`if (inTextBlock && currentBlockLines.length > 0 && !inCodeBlock)` → `if (inTextBlock && currentBlockLines.length > 0)`

2. 第 192 行：`else if (!foundFirstTask && !inCodeBlock)` → `else if (!foundFirstTask)`

3. 第 200 行：`if (!inCodeBlock && inTextBlock && currentBlockLines.length > 0)` → `if (inTextBlock && currentBlockLines.length > 0)`

4. 更新函数注释：明确说明"代码块内容作为普通文本块的一部分渲染"

## 修改文件

- `src/parser/index.ts`: 第 159、192、200 行，函数注释
