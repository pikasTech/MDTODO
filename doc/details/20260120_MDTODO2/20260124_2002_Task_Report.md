## R53.3 行内公式渲染修复

### 问题描述
任务块里面的行内公式（使用 `$...$` 格式）的渲染效果失效了。

### 根本原因
项目中使用了 `marked` 来渲染 Markdown 内容，但 `marked` 默认不支持 LaTeX 数学公式语法（`$...$` 和 `$$...$$`）。需要安装和配置 `marked-katex-extension` 和 `katex` 扩展来支持行内公式和块级公式的渲染。

### 解决方案
1. 安装 `katex` 和 `marked-katex-extension` 包
2. 在 `TaskList.tsx` 中配置 marked 使用 katex 扩展
3. 更新 webpack 配置以处理 katex 的 CSS 和字体文件
4. 更新 `package.json` 添加类型定义

### 修改的文件

1. **`package.json`**
   - 添加依赖：`katex`, `marked-katex-extension`
   - 添加类型定义：`@types/katex`

2. **`src/webview/components/TaskList.tsx`**
   - 导入 `markedKatex` 扩展和 katex CSS
   - 配置 marked 使用 katex 扩展，支持多种分隔符：
     - `$...$` - 行内公式
     - `$$...$$` - 块级公式
     - `\(...\)` - 行内公式
     - `\[...\]` - 块级公式

3. **`webpack.config.js`**
   - 添加字体文件处理规则（woff, woff2, eot, ttf, otf）
   - 将字体输出到 `fonts/` 目录

### 验证方式
在任务块中添加行内公式测试：
- 行内公式：`$E=mc^2$` 或 `\(E=mc^2\)`
- 块级公式：`$$f(x) = \int_{-\infty}^{\infty} \hat f(\xi)\,e^{2\pi i \xi x} \,d\xi$$` 或 `\[...\]`

### 注意事项
- bundle.js 大小会增加约 500KB（katex 库较大），这是预期的
- katex 字体文件会自动复制到 `resources/fonts/` 目录
