## R53.4 代码块渲染修复

### 问题描述
任务块里面的行内代码和跨行代码渲染不正常：
- 行内代码（使用 ` ` 包裹）没有渲染
- 跨行代码（使用 ``` ``` 包裹）渲染成了行内代码

### 根本原因
在 marked 17.x 版本中，`marked.parse()` 方法的第二个参数传递了 `{ async: false }` 选项，但这个选项在 marked 17.x 中可能不被支持或行为异常。

marked 17.x 的 `marked.parse()` 默认就是同步模式，不需要传递 `async: false` 选项。传递不支持的选项可能导致渲染器行为异常，从而影响代码块的正确解析和渲染。

### 解决方案
移除 `marked.parse()` 调用中的 `{ async: false }` 选项。

### 修改的文件

**`src/webview/components/TaskList.tsx`**

1. **EditableContentBlock 组件中的渲染逻辑**（约第1071行）：
   - 修改前：`marked.parse(content, { async: false })`
   - 修改后：`marked.parse(content)`

2. **TaskItem 组件中的 renderContent 函数**（约第1656行）：
   - 修改前：`marked.parse(task.title, { async: false })`
   - 修改后：`marked.parse(task.title)`

### 验证方式
在任务块中添加代码块测试：
- 行内代码：`` `code` ``
- 跨行代码：
  ````markdown
  ```javascript
  console.log('hello');
  ```
  ````

### 代码渲染原理
- **行内代码**：使用单个反引号 `` ` `` 包裹，如 `` `variable` ``，渲染为 `<code>variable</code>`
- **跨行代码**：使用三个反引号 ``` ```` ``` 包裹，可指定语言，渲染为 `<pre><code class="language-xxx">...</code></pre>`

marked 17.x 内置支持 GFM（GitHub Flavored Markdown），正确配置后应该能正常渲染这两种代码格式。
