### 2. 核心模块重构详情

#### 2.1 Parser 模块 (`cli/src/core/parser/index.ts`)

**变更前**: 使用 `vscode.workspace.fs.readFile()` 读取文件
**变更后**: 接受字符串内容，使用纯 Node.js `fs` 模块

**关键修改**:
- `parseFile()` 方法改为 `parseContent()`，接受字符串参数
- `parseFile()` 改为同步读取文件，使用 `fs.readFileSync()`
- 保留所有解析逻辑（tokenize、parseTask、calculateLinkStats 等）
- 保留链接存在性检查功能，使用 `fs.existsSync()`

**依赖关系**:
- 修改前: `import * as vscode from 'vscode';`
- 修改后: `import * as fs from 'fs'; import * as path from 'path';`

#### 2.2 FileService 模块 (`cli/src/core/services/fileService.ts`)

**完全重写**: 从零开始创建，不依赖任何 vscode API

**实现功能**:
- `readFile(filePath, options?)` - 读取文件
- `writeFile(filePath, content, options?)` - 写入文件
- `appendFile(filePath, content)` - 追加文件
- `exists(filePath)` - 检查文件是否存在
- `delete(filePath)` - 删除文件
- `stat(filePath)` - 获取文件状态
- `readDir(dirPath)` - 读取目录
- `mkdir(dirPath, recursive?)` - 创建目录
- `findTodoFiles(baseDir, pattern)` - 查找 TODO 文件

**使用 Node.js API**:
- `fs.readFileSync()`
- `fs.writeFileSync()`
- `fs.appendFileSync()`
- `fs.existsSync()`
- `fs.unlinkSync()`
- `fs.statSync()`
- `fs.readdirSync()`
- `fs.mkdirSync()`

#### 2.3 TaskManager 模块 (`cli/src/core/managers/taskManager.ts`)

**完全重写**: 从 `taskFileManager.ts` 重构，移除所有 vscode API 调用

**变更前**:
```typescript
vscode.window.showWarningMessage('请先打开一个TODO文件');
const content = await fileService.readFile(vscode.Uri.file(this.currentFilePath));
```

**变更后**:
```typescript
throw new Error('请先打开一个TODO文件');
const content = this.fileService.readFile(this.filePath);
```

**实现功能**:
- `getTasks()` - 获取所有任务
- `getTask(taskId)` - 获取指定任务
- `getAllTaskIds()` - 获取所有任务 ID
- `generateNewTaskId(parentId?)` - 生成新任务 ID
- `generateNewTaskContent(taskId, parentTaskId?)` - 生成新任务内容模板
- `addTask()` - 添加任务
- `addSubTask(parentTaskId)` - 添加子任务
- `deleteTask(taskId)` - 删除任务
- `updateTaskTitle(taskId, newTitle)` - 更新任务标题
- `markComplete(taskId)` - 标记为完成
- `markInProgress(taskId)` - 标记为进行中
- `markNotStarted(taskId)` - 标记为未开始
- `getStats()` - 获取统计信息
