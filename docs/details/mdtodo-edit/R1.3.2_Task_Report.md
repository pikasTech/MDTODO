## 二、subagent 可用性审查

### 2.1 subagent 使用场景分析

#### 适合 subagent 的场景
- `list` - 无需交互，直接列出任务
- `get <taskId>` - 无需交互，获取详情
- `complete <taskId>` - 无需交互，标记完成
- `start <taskId>` - 无需交互，开始任务
- `update <taskId> "<title>"` - 无需交互，更新标题
- `stats` - 无需交互，显示统计

#### 不适合 subagent 的场景
- `add` - 需要交互式输入标题
- `add-sub <parentId>` - 需要交互式输入标题

### 2.2 subagent 使用的关键要素

| 要素 | 状态 | 说明 |
|-----|------|------|
| 命令路径明确 | ✓ | `cd ~/.claude/skills/mdtodo-edit/scripts` |
| 绝对路径建议 | ✓ | 文档多次强调使用绝对路径 |
| 错误处理指南 | ✓ | 有调试步骤和常见错误表 |
| 任务 ID 格式 | ✓ | 清晰说明 `R1`, `R1.1` 格式 |
| skill.json | 缺失 | **未提及**，影响 skill 识别 |
| 部署说明 | 部分 | 只提到复制 SKILL.md |

### 2.3 缺失的关键文件

**问题**：没有创建 `skill.json` 文件。

Claude Code skill 正常工作需要以下文件：
```
~/.claude/skills/mdtodo-edit/
├── skill.json        # 必需：skill 元数据
├── SKILL.md          # 必需：skill 说明文档
└── scripts/          # 可选：CLI 脚本
```

`skill.json` 应当包含：
```json
{
  "name": "mdtodo-edit",
  "description": "Markdown TODO 任务管理工具",
  "commands": ["/mdtodo"]
}
```
