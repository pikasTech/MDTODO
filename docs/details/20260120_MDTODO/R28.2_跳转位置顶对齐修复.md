# R28.2 跳转位置顶对齐修复

## 问题描述

跳转下拉菜单跳转到任务时，位置是居中的，导致很长的任务看不到开始的位置。用户希望顶对齐，例如跳转到R4，那么R4的开始位置就在顶部。

## 修改内容

### 文件：`vscode-mdtodo/src/webview/components/TaskList.tsx`

修改了3处 `scrollIntoView` 调用，将 `block: 'center'` 改为 `block: 'start'`：

1. **第160行**（pendingScrollTaskId useEffect 中）
   - 用途：添加新任务后滚动到任务位置
   - 修改：`taskElement.scrollIntoView({ behavior: 'smooth', block: 'center' });`
   - 改为：`taskElement.scrollIntoView({ behavior: 'smooth', block: 'start' });`

2. **第573行**（handleJumpToTask 函数中）
   - 用途：跳转下拉菜单跳转到指定任务
   - 修改：`taskElement.scrollIntoView({ behavior: 'smooth', block: 'center' });`
   - 改为：`taskElement.scrollIntoView({ behavior: 'smooth', block: 'start' });`

3. **第593行**（handleScrollToTask 函数中）
   - 用途：VSCode编辑器滚动同步到Webview
   - 修改：`taskElement.scrollIntoView({ behavior: 'smooth', block: 'center' });`
   - 改为：`taskElement.scrollIntoView({ behavior: 'smooth', block: 'start' });`

## 验证

运行 `npm run compile` 编译成功，bundle.js 已更新。

## 效果

- 跳转下拉菜单：跳转到任务时，任务标题会显示在可视区域顶部
- 新任务滚动：添加新任务后，新任务会显示在可视区域顶部
- 双向滚动同步：从VSCode编辑器点击跳转到Webview时，任务会显示在顶部
