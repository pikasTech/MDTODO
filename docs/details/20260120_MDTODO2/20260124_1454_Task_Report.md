# R51.16 任务报告 - 输入框高度与图标一致

## 任务描述
跳转，搜索，筛选的输入框高度和图标不一致，调高一点和图标一致。

## 问题分析
- 图标按钮 `.toolbar-icon-btn` 高度：padding: 6px * 2 + svg 14px = 26px
- 输入框 `.header-search-input` 高度：padding: 4px * 2 + line-height ≈ 20px，视觉上偏矮
- 下拉框 `.header-select` 和 `.filter-dropdown` 高度：padding: 4px * 2 + line-height ≈ 20px

## 修复内容
修改 `src/webview/components/TaskList.css`：

1. **`.header-select`**（跳转下拉框）
   - padding: 4px → 6px（上下）
   - 添加 height: 26px

2. **`.header-search-input`**（搜索输入框）
   - padding: 4px 8px → 6px 8px（上下）
   - 添加 height: 14px 和 line-height: 14px

3. **`.filter-dropdown`**（筛选下拉框）
   - padding: 4px → 6px（上下）
   - 添加 height: 26px

## 验证结果
- 编译成功，bundle.js 已更新
- 输入框、下拉框高度现在与图标按钮一致（约 26px）
