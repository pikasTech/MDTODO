# R51.8 任务报告

## 任务描述

跳转下拉菜单在完成一次跳转后，应当复位到"跳转"状态，而不是保持被定位的ID。

## 修改文件

- `src/webview/components/TaskList.tsx`

## 修改内容

### 位置

`handleJumpToTask` 函数（约第639-657行）

### 修改说明

在 `handleJumpToTask` 函数中，原有逻辑在用户选择任务后设置 `jumpToTaskId` 为所选任务ID，但跳转完成后并未复位下拉菜单。

修改后在滚动完成时（`setTimeout` 回调中）添加了 `setJumpToTaskId('')` 调用，将下拉菜单复位到"跳转"状态。

### 代码变更

```typescript
// 修改前：跳转完成后保持 jumpToTaskId 不变
setTimeout(() => {
  const taskElement = document.querySelector(`[data-task-id="${taskId}"]`);
  if (taskElement) {
    taskElement.scrollIntoView({ behavior: 'smooth', block: 'start' });
  }
}, 50);

// 修改后：跳转完成后复位下拉菜单
setTimeout(() => {
  const taskElement = document.querySelector(`[data-task-id="${taskId}"]`);
  if (taskElement) {
    taskElement.scrollIntoView({ behavior: 'smooth', block: 'start' });
    // 【R51.8】跳转完成后复位下拉菜单到"跳转"状态
    setJumpToTaskId('');
  }
}, 50);
```

## 验证方式

1. 打开跳转下拉菜单
2. 选择任意任务ID进行跳转
3. 跳转完成后，观察下拉菜单是否自动复位显示"跳转"选项
